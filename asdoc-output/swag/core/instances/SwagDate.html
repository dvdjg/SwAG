Implements swag.interfaces.core.instances.ISwagDate

Contains and extends a standard Flash Date object with various formatting and comparison options as well as other date-related utilities.

This instance is intended to be used alone but is also used throughout the SwAG toolkit for date support.

dateIndexproperty 
dateIndex:uint  [read-only]

An unsigned integer value based on the date of the instance in the format YYYYMMDD.

This allows the date object to be used as a unique index value in sorted data sets like numeric arrays or as a unique key in databases.

Numbers in this value that are less than 10 are padded with an extra 0 to ensure that index valued and data length are retained (i.e. the returned value will always be 8 digits long).

MSDOSDateproperty 
MSDOSDate:uint  [write-only]

Sets the date (day / month / year) value for the SwagDate instance from a 16-bit packed MS-DOS time format.

If this value is combined with a 16-bit time value, as with most standard packed MSDOS date/time stamps, it can be isolated using a simple shift-right operation: 

MSDOSDateValue=MSDOSPackedTimeDateValue >> 15; //Shift right by 15 bits

Because Flash uints are 32 or posisbly 64 bits, only the last 16 bits will be evaluated. This value is assumed to be stored in a MSB (Most Significant Bit first) order.

public function getDaysPerMonth(... args):uint

Returns the number of days of the specified month, or of the current month associated with the SwagDate instance if none specified.

Parameters

... args — The month number to evaluate. This parameter may either be a numberic type (starting at 0 for January), a string (valid entries include the full month name, month abreviation, or numeric string representing the month number), or omitted in which case the current SwagDate month value will be used.

Returns
uint — The number of days in the specified month. Note: The value for February always returns a non-leap year number (29 days). Use the isLeapYear method to adjust this value wherer required.

public static function getMonthName(monthIndex:uint, ... args):String

Returns the full or partial name of the month represented by the input parameter.

Parameters

monthIndex:uint — The numeric index of the month, starting with 0=January, 1=February, 2=March, etc.
 
... args — Optional parameters include (in order):

  • abbrev (Boolean) - If true, the three-letter abbreviation for the month name is returned instead of the full name (e.g. "Jul" instead of "July"). Default is false.
  • lowercase (Boolean) - If true, the month name (full or abbreviated) is returned as a lowercase string instead of a mixed-case string in which the first letter is capitalized. Default is false.

Returns
String — The name of the month, either full or abbreviated / capitalized or lowercased (depending on the optional parameters), of the specified month number.
getMonthNumber()method 
public static function getMonthNumber(monthName:String):uint

Translates the specified month string (name or numeric string representation), to its 0-based month value.

Months are numbered starting from 0=January, 1=February, 2=March, and so on.

Parameters

monthName:String — The name, 3-letter abreviation (with an optional period at the end), or numeric string representation of the month. For example, "February", "Feb.", "february", "feb", and "1", are all equivalent and will return month number 1.

Returns
uint — The 0-based month number represented by the parameter. That is, 0=January, 1=February, and so on.

isLeapYear()method 
public static function isLeapYear(... args):Boolean

Evaluates the specified year to determine whether or not it's a leap year.

Use this in conjunction with any methods that evaluate the number of days in a specific month (especially February).

Parameters

... args — The year value to evaluate. This may either be a numeric type, which will be used as is, a string, which will be converted to a numeric type, or a SwagDate instance who's year property will be used.

Returns
Boolean — True if the specified parameter is a valid leap year, false if it's not or if there was an error converting the parameter to a usable (i.e. numeric) type.
