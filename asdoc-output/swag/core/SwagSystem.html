Implements swag.interfaces.core.ISwagSystem

Provides or collects a variety of methods and properties that can be used by the developer to deeply inspect and (sometimes) update the host system settings (the Flash / AIR version, the operating system, browser, etc.), or various core ActionScript objects.

settingsproperty 
settings:Object  [read-only]

Contains all of the settings detected by SwagSystem.

As well as including all of the properties from the flash.system.Capabilities, flash.system.System, and flash.desktop.NativeApplication classes, the settings object also includes the following data:

  • isAIR (Boolean) - True if the enclosing runtime is AIR.
  • isWeb (Boolean) - True if the enclosing runtime is the Flash web player (plugin or ActiveX).
  • isStandalone (Boolean) - True if the enclosing runtime is the standalone Flash player.
  • isMobile (Boolean) - True if the enclosing runtime is a mobile version of AIR or the Flash web player.

Any properties that are not available are set to null; for example, when attempting to read AIR properties from within the Flash web player.

onExists()method 
public static function onExists(property:String, targetObject:*, callbackMethod:Function, allowNull:Boolean = true):Boolean

Begins monitoring for the existence of a property within a specific object, and invokes a method when that property becomes available (is not undefined and, optionally, is not null).

Null objects are considered to exist and will trigger the callbackMethod unless the allowNull parameter is false.

Parameters

property:String — The property to begin monitoring for. Must not be null or an empty string.
 
targetObject:* — The object that is expected to contain the property at some time in the future. Must not be null.
 
callbackMethod:Function — The method to invoke when the specified property within the targetObject becomes available. The callback method should not expect any parameters.
 
allowNull:Boolean (default = true) — If true, null objects are considered to exist and will invoke the callbackMethod. If false, only non-null and non-undefined objects are considered to exist.

Returns
Boolean — True if the property monitor was successfully created and started. False is returned if there was a problem creating the watch monitor (for example, if one of the supplied parameters is incorrect).
